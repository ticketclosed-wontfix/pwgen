Your passwords never leave your browser

This tool is designed with a zero-trust architecture. All passphrase generation happens entirely in your browser using client-side JavaScript. The web server delivers static HTML — it performs no computation, stores no data, and has no access to anything you generate.

🧠 Client-side only

All generation logic runs in your browser. The server serves a single static HTML file and nothing else.

🚫 Zero data storage

No cookies, no local storage, no analytics, no tracking. Nothing is stored or transmitted.

📡 No API calls

Open your browser's Network tab — you'll see zero outbound requests during generation. Try it.

📖 Fully auditable

The complete source is published on GitHub. You can verify every line of code yourself.

Verify what's running

Since this is a purely client-side application, verification is straightforward — the code your browser executes is the same static HTML file served by the server. Here's how to confirm it:

1

View the source

Right-click this page → View Page Source (or press Ctrl+U). What you see is the entire application. There is no server-side code.

2

Check the network

Open DevTools → Network tab → generate a passphrase. You'll see zero outbound requests. The word list, substitution logic, and UI are all embedded in the HTML.

3

Compare with GitHub

The source is published at github.com/ticketclosed-wontfix/pwgen. The index.html served here is the same file in the repository. Compare them directly.

4

Self-host it

Clone the repo, open index.html in your browser. It works without any server at all — just double-click the file. No build step, no dependencies.
